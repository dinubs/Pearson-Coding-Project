About

This project is for the Pearson Coding Project. The main idea is to create something using the Pearson APIs to enhance student learning. I plan on using Ruby on Rails to create this, seeing as it is the easiest to implement a database, and add a lot to the overall site.

This specific project will implement the Financial Times API, and the Dictionary API. The app that I plan to create is to be used for research for student projects, or just overall learning, after looking at the Financial Times website, and seeing how difficult it is to navigate, but seeing how helpful the content could be for students to use, in their projects, I decided to create an app that will allow students to do just that.

As a student I know what is like to find resources for research, especially current ones, the articles that I looked at on Financial Times all were extremely interesting and I feel that this information could be implemented in a better way, which I plan to do.

Plan

My plan for creating this project is to first implement a way to view the text from all 40,000 articles provided by Financial Times by adding the id into my own database for faster app speed, otherwise, I would be making multiple requests, and I can provide more options to students to find things this way.

Second I want to implement a way to use the Pearson Dictionary API so that when students are reading through an article and come across a word that they don't understand or don't know, they can simply highlight the word and the app will provide a definition for them, or a list of synonyms. This way students can understand what they are reading about without having to pull out a dictionary, or go to an external website. Along with the Dictionary API I plan to implement a way for to students to cite the website on the page.

Third I plan on implementing a user system so that students can either upvote or downvote an article, the more upvotes a post has the more likely it is to be found on the front page. This is to showcase popular articles that otherwise would go unlooked if users didn't vote on them.
