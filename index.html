Cabretio

This project started off as a way to make it easier to read articles from www.ft.com and has since evolved into being a full research studio for students.

Formerly known as The FT Researcher, Cabretio is now a full research studio for students, with 2,529 articles on the site to start off, and 2,522 of those articles coming from www.ft.com. Once a user signs up they can add articles from almost any website to grow the system even more. I've created some innovative technology for students when browsing the website, these including a way to save your works cited citations onto Cabretio, a way to download any article on the website into a word document, and last but not least dictionary that will define any word on the website.

Citations

The biggest problem that I consistently run into when writing a research paper is the works cited page, so to solve this I built a way to let students save any web page into a citation, by providing the bare minimals to create an MLA citation. These citations are then categorized into which ever way the student wants (e.g. student can save all english paper citations in one spot, and all history citations into one spot).

Being able to save a citation is one thing that other research databases don't allow you to do, yes you can download them, but being able to save them from a school computer, then being able to see them at home is something different.

Word Documents

A word document is one thing that every student knows, and knows extremely well by college. I've used some very interesting technology to allow students to be able to download any article from Cabretio, along with being able to download all citations that a student creates, making a works cited one of the easiest things to do.

Word documents, make it extremely easy to download an article in whichever style the student wants, and then being able to annotate and highlight the article without having website styles getting in the way.

Dictionary

Last but not least, is the API that I implemented into Cabretio, a dictionary is one of the most important things any student can have, but the problem is that it's extremely difficult to get a definition easily without having to leave their current page. I solved this by allowing users to be able to simply highlight whichever word they want, and instantly the definition of the highlighted word comes up, allowing you to easily continue reading the article, and now knowing what that word means.

Along with the dictionary I've wrote a background script to save www.ft.com articles into Cabretio's database, so if I ever wanted to add more I easily could.

Future

Just because the competition is coming to an end, does not mean that I'm finished working on Cabretio, I feel that I can grow Cabretio to actually be something. I have extra things that I'm working on, that I don't have entirely finished right now, but they'll be out soon. I'm working on an iPhone app to go along with Cabretio, to allow you to read articles on you iOS device along with everything else the web site can do. Also a Google Chrome Extension to allow you to save any page as a word document, or get the citation right there, without having to visit Cabretio at all (although you still should, because it's awesome and on the website you can do more than the Chrome Extension.

It's been a long couple of months working on this web site, but it's been extremely enjoyable, and I'm glad I was able to be a part of this competition.
